# 파일럿 실행 가이드 (v1, 2026-04-21)

> 본 가이드는 전체 Phase 1 풀런에 앞서, **파이프라인 무결성** 과
> **단일 (agent, model) 독립 실행 정책** 이 실제로 동작하는지 확인하기 위한
> 최소한의 smoke run 절차입니다.

## 목표

1. 조건별 `setup.ps1` 이 workspace 에 올바른 문서만 배치하는지
2. 어댑터(`agents/*.ps1`) 가 CLI 를 정상 호출하여 `stream.jsonl` 을 남기는지
3. 품질 게이트(pytest / ruff / mypy) 가 결과 디렉터리에 `pytest.json` 등을 남기는지
4. `evaluate.py` 가 `metrics.json` 을 산출하는지
5. `aggregate.py` 가 `summary.csv` + `report.md` 를 생성하는지

## 준비

```powershell
# 1) 가상환경 & 파이썬 의존성
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

# 2) 사용하려는 에이전트 CLI 1개만 설치 (파일럿은 Cursor 권장)
irm 'https://cursor.com/install?win32=true' | iex
$env:CURSOR_API_KEY = "sk-..."   # 본인 키

# 3) 선택: quota monitor 별도 터미널에서 기동
# (파일럿은 2 run 밖에 안 되어 생략 가능)
```

## 파일럿 1 — Dry run (에이전트 호출/게이트 생략)

먼저 **로직만** 검증합니다. 실제 토큰 소비 없음.

```powershell
.\experiments\runner\run_experiment.ps1 `
    -Agent cursor -Model sonnet `
    -Conditions "C0,C1" -Repeats 1 -Tasks "task-1-todo-crud" `
    -DryRun -NoEvaluate
```

검증 포인트:
- 콘솔에 `matrix size : 2 run` 출력
- `experiments/results/<ts>/cursor/sonnet/runs.csv` 생성
- `experiments/results/<ts>/cursor/sonnet/C0/rep1/task-1-todo-crud/stream.jsonl` 에 `{"event":"dry_run"}` 기록

## 파일럿 2 — C0 만 실제 호출 (에이전트 1회)

토큰을 최소한만 쓰면서 **어댑터 → 게이트 → evaluate → aggregate** 까지 실제로 돌립니다.

```powershell
.\experiments\runner\run_experiment.ps1 `
    -Agent cursor -Model sonnet `
    -Conditions "C0" -Repeats 1 -Tasks "task-1-todo-crud"
```

확인:
- `stream.jsonl` 이 비어 있지 않고 JSONL 이벤트가 여러 줄
- `pytest.json`, `ruff.json`, `mypy.txt` 존재
- `metrics.json` 에 8개 지표 필드 존재
- `summary.csv` 한 행, `report.md` 생성

만약 `pytest` 가 "no tests collected" 로 종료하면, 어댑터는 `tests/` 폴더조차 못 만든 것입니다. 이는 C0 (docs 없음) 의 **의도된 결과** 이며, `build_success=false`, `test_pass_rate=0` 으로 기록됩니다.

## 파일럿 3 — C1 만 (요구사항 문서 1개) 실제 호출

```powershell
.\experiments\runner\run_experiment.ps1 `
    -Agent cursor -Model sonnet `
    -Conditions "C1" -Repeats 1 -Tasks "task-1-todo-crud"
```

C0 과 비교했을 때 `requirements_fulfillment` 와 `test_pass_rate` 가 개선되어야 합니다. 만약 동일하거나 더 나쁘다면, adapter 가 워크스페이스에 접근 못하고 있거나 프롬프트가 전달되지 않은 것입니다. `stream.jsonl` 에 `user` / `assistant` 턴 수를 확인하세요.

## 파일럿 4 — 다른 에이전트 1개로 sanity check

Cursor 이외의 에이전트도 어댑터가 잘 작동하는지 확인합니다. Aider 를 추천 (pip install 로 가볍게 검증 가능).

```powershell
pip install aider-chat
$env:OPENAI_API_KEY = "sk-..."

.\experiments\runner\run_experiment.ps1 `
    -Agent aider -Model "openai/gpt-4o-mini" `
    -Conditions "C1" -Repeats 1 -Tasks "task-1-todo-crud"
```

- `agent.meta.json` 의 `agent` 가 `"aider"` 인지
- `ws/aider-openai_gpt-4o-mini-C1-rep1/` 에 파일 수정 이력이 있는지 (git status)

## 파일럿 5 — 같은 RunId 로 두 에이전트 병합 집계

```powershell
$id = Get-Date -Format 'yyyyMMdd_HHmmss'

.\experiments\runner\run_experiment.ps1 -RunId $id -Agent cursor -Model sonnet       -Conditions "C0,C1" -Repeats 1 -Tasks "task-1-todo-crud"
.\experiments\runner\run_experiment.ps1 -RunId $id -Agent aider  -Model "openai/gpt-4o-mini" -Conditions "C0,C1" -Repeats 1 -Tasks "task-1-todo-crud"

# 교차 집계는 두 번째 호출 끝에 이미 갱신되지만, 명시적으로도 가능:
python .\experiments\runner\aggregate.py --run-root .\experiments\results\$id
Get-Content .\experiments\results\$id\report.md | Select-Object -First 50
```

`report.md` 의 "요구사항 충족률 히트맵" 에 2×2 (agent×cond) 행렬이 찍히면 OK.

## 파일럿 체크리스트

- [ ] Dry run 2건 매트릭스 매핑 OK
- [ ] `stream.jsonl` 이 JSONL 형식(한 줄에 한 JSON)으로 작성됨
- [ ] `agent.meta.json` 의 `wall_seconds` > 0
- [ ] `pytest.json` 의 `summary.total` > 0 (C1 이상에서)
- [ ] `metrics.json` 의 `requirements_fulfillment` 가 0~1 사이
- [ ] `summary.csv` 가 UTF-8 BOM 포함으로 저장됨 (Excel 호환)
- [ ] `report.md` 에 한글 헤더 깨짐 없음
- [ ] C0 < C1 에서 최소 1개 이상 지표가 개선됨 (방향성)

## 파일럿 실패 시 트러블슈팅

| 증상 | 원인 후보 | 조치 |
|---|---|---|
| `adapter not found` | Agent 철자 오타 | `-Agent cursor|codex|aider|copilot|custom|manual` 확인 |
| `cursor-agent: not recognized` | CLI 미설치 또는 PATH 누락 | 셸 재시작, `where.exe cursor-agent` |
| `ModuleNotFoundError: yaml` | `requirements.txt` 미설치 | `pip install -r requirements.txt` |
| `pytest` 0 collected | 에이전트가 `tests/` 를 만들지 못함 | `stream.jsonl` 에서 tool_use 수 확인, 프롬프트 수신 확인 |
| `.stop` 파일이 반복적으로 생성 | `monitor_quota.ps1` 가 실패율 초과로 판단 | 임계값 완화(`-FailureThreshold 10`) 또는 수동 삭제 |
| 한글 파일명 에러 (`요구사항/`) | Windows UTF-8 설정 | PowerShell 7 권장, `chcp 65001` |
