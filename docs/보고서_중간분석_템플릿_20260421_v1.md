## 5. 정성 분석

### 5.1 실패 사례 (대표 3개)

각 사례별로: `RunId`, `agent/model/C*/rep/task`, `metrics.json` 요약, 스트림 하이라이트, 원인 추정.

- 사례 1: `cursor/sonnet/C0/rep1/task-2-jwt` — 가설: docs 부재로 JWT secret 관리가 `os.environ` 대신 하드코딩. 영향: `qual_layering=0`, `fr_login=0`.

### 5.2 성공 사례

- 사례 A: `cursor/gpt-5/C3/rep2/task-2-jwt` — 설계-구현 일치도 0.95. `docs/api.md` 의 `/auth/login` 스펙을 90% 이상 재현.

## 7. 위협 요인 (Threats to Validity)

- **내적 타당도**: LLM-judge 편향 → 20% 샘플 수동 검수 결과 `<n>/<m>` 일치.
- **외적 타당도**: FastAPI TODO 도메인 한정. SWE-bench Lite (Phase 2) 에서 일반화 검증 예정.
- **구성 타당도**: 지표 #7(manual_fix_proxy)은 stream 이벤트 기반 프록시라 실제 사용자 수정을 과소/과대 추정 가능.
