# Phase 1 풀런 실행 가이드 (v1, 2026-04-21)

> 파일럿이 성공한 전제 하에 **여러 (agent, model) 을 순차 독립 실행** 하여 전체 매트릭스를 수집하는 절차입니다.
> 같은 `RunId` 로 묶어 마지막에 **교차 집계** 합니다.

## 설계 원칙

1. **Stand-alone 실행**: 1회 호출 = 1개 (agent, model). 여러 모델 동시 실행 금지.
2. **구독별 잔여 크레딧 소진**: Cursor Pro / Codex CLI / Aider(OpenAI·Anthropic) / Copilot 을 각각 돌려 잔액을 분산 소진.
3. **환경 통제**: 한 셸 세션, 한 Python 가상환경, 동일 `seed_prompt.md` 와 `acceptance.*.yaml` 를 사용.
4. **안전장치**: `monitor_quota.ps1` 를 별도 터미널에서 띄워 실패율/시간 예산 초과 시 `.stop` 생성.

## 매트릭스 (한 (agent, model) 당)

```
5 조건 × 3 반복 × 3 task = 45 run
workspace 개수: 5 × 3 = 15  (C4 는 rep 당 3 task 공유)
```

## 실행 절차

### 1) RunId 고정

```powershell
$runId = Get-Date -Format 'yyyyMMdd_HHmmss'
"runId=$runId" | Set-Content .\experiments\results\CURRENT_RUN.txt
```

### 2) Quota monitor (별도 터미널)

```powershell
.\experiments\runner\monitor_quota.ps1 `
    -RunRoot (Join-Path (Resolve-Path .\experiments\results) $runId) `
    -TimeBudgetMinutes 360 `
    -FailureThreshold 6
```

- `-TimeBudgetMinutes 360`: 6시간 예산
- `-FailureThreshold 6`: 최근 10 run 중 6 run 이상이 adapter_exit != 0 이면 중단
- `.stop` 파일이 생기면 runner 가 다음 iteration 시작 전에 `"Quota stop file detected"` 로그 후 중단

### 3) (agent, model) 별 풀런 — 순차 실행

## 중간 점검

3.1 이 끝난 시점마다 아래 지표를 확인합니다 (같은 방식으로 3.2~3.6 반복).

```powershell
# 최근 10 run 평균
python -c @"
import json, statistics, pathlib
rows = [json.loads(p.read_text(encoding='utf-8')) for p in pathlib.Path(r'$((Resolve-Path .\experiments\results\$runId).Path))').rglob('metrics.json')]
print('n=', len(rows))
for key in ['requirements_fulfillment','test_pass_rate','design_alignment','static_errors_total','reprompt_count','wall_seconds']:
    vals = [r.get(key) for r in rows if r.get(key) is not None]
    if vals:
        print(f'{key:28s} mean={statistics.mean(vals):.3f}  n={len(vals)}')
"@
```

`requirements_fulfillment` 가 조건별로 단조 증가 경향(C0 < C1 < C2 ≤ C3) 인지 확인.

## 중단/재개

### 중단

- `Ctrl+C`: runner 는 현재 run 을 완료한 뒤 종료 (pytest/evaluate 까지 진행).
- `.stop` 파일 생성: quota monitor 또는 수동.
  ```powershell
  ni (Join-Path (Resolve-Path .\experiments\results\$runId) .stop) -ItemType File
  ```

### 재개

동일 `-RunId $runId` 와 필요한 조건/반복만 지정해 재호출.
`setup.ps1` 이 workspace 를 조건별로 초기화하므로 오염 걱정 없음.
이미 완료된 run 은 덮어쓰여지지만 `runs.csv` 는 추가되어 보존됨.
